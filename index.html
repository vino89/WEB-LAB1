Catégorie 1 : HTML

Paragraphe
Ce site permet à son utilisateur de gérer les tâches à accomplir pour le premier labo du cours de WEB 2023 de la filière informatique de la HEIG. Typiquement chaque tâche aura une case qui peut être cochle afin de gérer l'avancée du projet. Un travail qui s'est révélé totalement inutile, étant donné que les cases ne garde pas leur état, ce qui finira par pousser le créateur de la page vers l'alcoolisme, la dépression et finalement... le développement logiciel.

Écrire en souligné
Parmi les taches les plus simples il est demandé d'avoir au moins un morceau de texte souligné, ce qui est fait en utilisant la balise <u>.
Écrire en gras
La seconde tâche abordable, consiste à écrire une partie du texte en gras, à l'aide de balise <b>.
Écrire en italique
Finalement, la troisième tâche format de texte abordable est de présenter une partie du texte en italique en utilisant les balises <i>
Ajouter un lien
La balise <a> est comme un portail magique dans le monde du web. Par exemple un lien menant droit vers un guide d'utilisation des checkbox transporté vers de nouvelles aventures. C'est comme si chaque lien était un billet pour un voyage excitant, sans besoin de faire la queue.

Catégorie 2 : CSS

Le projet contient au moins un fichier *.css
Notre projet HTML doit rester à la pointe de la mode web, donc il est impératif d'avoir au moins un fichier CSS. C'est comme la garde-robe secrète qui donne à nos pages un look d'enfer. Après tout, même les pages web veulent être à la mode, non ?
Ajouter et utiliser au moins 2 pseudo classes
Ajouter du style à nos éléments avec au moins deux pseudo-classes est comme donner une cravate élégante à notre HTML. Les pseudo-classes sont comme les accessoires de mode, mais pour les balises HTML. Qui a dit que le web ne pouvait pas être à la mode ?
Le layout Flexbox est utilisé au moins une fois
Intégrer au moins une fois le layout Flexbox, c'est comme donner à nos éléments une séance d'entraînement. Ils peuvent se tordre et se plier pour s'adapter à toutes les tailles d'écran. La gymnastique de l'agencement HTML, un entraînement pour les éléments en quête de flexibilité !
Le layout Grid est utilisé au moins une fois
Utiliser au moins une fois le Grid Layout, c'est comme créer un plan stratégique pour nos éléments. Imaginez-les comme des troupes bien organisées, alignées sur une grille. Parce que même les éléments HTML ont besoin d'une stratégie de bataille.
Déclarer et utiliser au moins 2 classes
Intégrer et déclarer au moins deux classes, c'est comme donner une personnalité à nos éléments HTML. Ils peuvent rejoindre des groupes sociaux, comme les divas CSS ou les rebelles JavaScript. Les classes - parce que même le code HTML a son côté social.

Introduire au moins un fichier JavaScript, c'est comme embaucher un magicien pour ajouter une touche d'interactivité. Les fichiers JavaScript sont les magiciens du code, faisant des tours pour impressionner nos visiteurs.
Au moins une variable est utilisée
Utiliser au moins une variable, c'est comme donner une identité à nos éléments HTML. Les variables sont les héros méconnus du code, travaillant dans l'ombre pour rendre notre page web plus dynamique. Après tout, même le code a besoin de ses propres héros.
Au moins une fonction est utilisée
Introduire au moins une fonction, c'est comme engager un architecte pour construire des structures dans notre code. Les fonctions sont les architectes du code, créant des plans pour organiser et simplifier notre projet. Parce que même le code a besoin d'une bonne architecture.
